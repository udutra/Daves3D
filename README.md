# daves3d

Especialização em Desenvolvimento de Jogos Digitais

Programação 3D & Redes Aplicadas a Jogos - 2017/2

O trabalho consiste no desenvolvimento de um jogo multiplayer em 3D com as seguintes características:

    (X) Arquitetura servidor e clientes, utilizando as bibliotecas de rede da Unity.
    (X) O jogo possui dois estados principais: lobby ou playing.
    (X) Os jogadores podem se conectar ao servidor em qualquer estado.
    (X) Durante o lobby, os jogadores falam por chat de texto e indicam quando estão prontos para jogar.
    (X) Quando todos estiverem prontos, o jogo começa (vai para o estado playing).
    (X) Criar um sistema de pontos para determinar quem foi o vencedor (competitivo) ou se os jogadores venceram ou não o desafio (cooperativo);
    (X) Cada partida deve durar no máximo 5 minutos e mostrar os resultados durante 10 segundos.
    (X) Depois de exibir os resultados, o jogo deve voltar para o estado de lobby.
    (X) Jogadores que conectarem durante o estado playing assistirão a partida no modo “espectador” e só poderão jogar efetivamente na próxima partida.
    (X) Se o servidor desconectar, o jogo encerra para todos clientes.
    (X) Apresentar o estado do jogo para todos os jogadores conectados de forma íntegra e eficiente.
    (X) Suportar até 4 jogadores jogando simultaneamente.
    (X) Exibir mensagens de desconexão e problemas de rede.
    (X) Preencher o nome do jogador.
    (X) Selecionar o modelo do jogador (customizar cores ou forma).
    (X) Opção “host”.
    (X) Opção “join”.
    (X) Opção “quit game”.
    (X) Câmera “1st person” ou “3rd person”.
    (X) Colisão funcional com todos os objetos do jogo.
    (X) Ter um mapa feito exclusivamente para o jogo.
    (X) Deve ter mais de uma elevação (mais de um plano de ação).
    (X) Texturizado e iluminado apropriadamente.
    (X) Utilização de sons e efeitos nas ações dos jogadores.
    (X) Implementar signs & feedbacks apropriados.
